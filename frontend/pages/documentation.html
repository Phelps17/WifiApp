Adding Local Wifi Connect Library to Your Android Project

Adding the Local Wifi Connect Library to your Android application can provide a plethora of services to both you and your customers. Lucky for you, it is also incredibly easy to add to a new or existing project and begin using.

Customers with your application installed will be automatically authenticated at your registered networks when they become geographically close enough to join, regardless of how often you change your network passwords. This allows your business to keep it's visitor networks more secure by allowing you to change passwords often. Businesses are also able to track metrics and run analytics on new and return visitors to their locations to provide better insite on their customer base. Additionally, You can also view a list of MAC Addresses of the devices that have connected to your network. All of these features are easily accessable from the Local Wifi Connect Admin Portal.

Set Up Google Play Services

To access location data, your app’s development project must include Google Play Services. Download and install Google Play services component via the SDK Manager and add the library to your project. For details, see the guide to Setting Up Google Play Services.

Specify App Permissions

Apps that use location services must request location permissions. Android offers two location permissions: ACCESS_COARSE_LOCATION and ACCESS_FINE_LOCATION. The permission you choose determines the accuracy of the location returned by the API. If you specify ACCESS_COARSE_LOCATION, the API returns a location with an accuracy approximately equivalent to a city block.

This lesson requires only coarse location. Request this permission with the uses-permission element in your app manifest, as the following code snippet shows:

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.google.android.gms.location.sample.basiclocationsample">

    <uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
</manifest>

Set Up Your Google Service ID

After running Gradle you can navigate to res/values and open a file called google_maps_api.xml. Open the web address in that file to register your app for Google Play Services. Once you are registered, you will receive a Google Play authentication key starting with “AIza”. Replace the "google_maps_key" with the key you receive.

Set Up Local Wifi Connect Android Library

Import the Lcal Wifi COnnect library to your class and create a class variable using LocationServcieController lsc;.

In your activity’s onCreate() method, create an instance of LocationServiceController as the following code snippet shows. 

this.lsc = new LocationServiceController(this);

Adjust your onStart() and onStop() methods as follows:

protected void onStart() {
    lsc.connect();
    super.onStart();
}

protected void onStop() {
    lsc.disconnect();
    super.onStop();
}
